## Примечания

1. **Безопасность:**
   - Все пароли хранятся в виде bcrypt хешей в базе данных
   - В продакшене необходимо изменить все пароли на более сложные
   - Рекомендуется использовать пароли длиной не менее 12 символов

2. **Инициализация:**
   - Пользователи создаются автоматически при запуске скрипта `backend/init_users.py`
   - Скрипт проверяет наличие пользователей и не создает дубликаты

3. **Доступ инженеров:**
   - По умолчанию инженеры не имеют доступа к оборудованию
   - Доступ предоставляется операторами или администраторами через API:
     - `POST /api/users/{user_id}/equipment-access`
   - Или через веб-интерфейс (если реализовано)

4. **Смена паролей:**
   - В текущей версии смена паролей через API не реализована
   - Для смены пароля необходимо обновить запись в базе данных напрямую

## Создание новых пользователей

Новых пользователей можно создать через API:
```bash
POST /api/users
Authorization: Bearer {token}
Content-Type: application/json

{
  "username": "new_user",
  "email": "new_user@example.com",
  "password": "secure_password",
  "full_name": "Новый Пользователь",
  "role": "engineer",
  "is_active": true
}
```

Требуется право `manage_users` (только для admin и chief_operator).
