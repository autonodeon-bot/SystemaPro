# Статус инициализации и сборки

## Выполнено

### 1. Скрипт инициализации пользователей
- ✅ Создан файл `backend/init_users.py`
- ✅ Скрипт создает 5 тестовых пользователей:
  - `admin` / `admin123` (Администратор)
  - `chief_operator` / `chief123` (Главный оператор)
  - `operator` / `operator123` (Оператор)
  - `engineer1` / `engineer123` (Инженер 1)
  - `engineer2` / `engineer123` (Инженер 2)

**Запуск:**
```bash
cd backend
python init_users.py
```

**Примечание:** Скрипт требует подключения к базе данных. Убедитесь, что:
- Backend контейнер запущен
- Переменные окружения для БД настроены
- Таблицы созданы (выполняется автоматически при запуске backend)

### 2. Сборка мобильного приложения
- ✅ Создан скрипт `check-and-build.bat` для автоматической проверки и сборки
- ✅ Создан скрипт `mobile/build-app.bat` для сборки APK

**Запуск сборки:**
```bash
cd mobile
.\build-app.bat
```

Или автоматически:
```bash
.\check-and-build.bat
```

**Результат:**
- APK файл: `mobile/build/app/outputs/flutter-apk/app-release.apk`

## Устранение проблем

### Если скрипт инициализации не работает:
- Проверьте подключение к БД
- Убедитесь, что backend контейнер запущен
- Проверьте переменные окружения (DB_USER, DB_PASS, DB_HOST, etc.)

### Если сборка APK не работает:
- Убедитесь, что Flutter установлен: `flutter --version`
- Установите зависимости: `flutter pub get`
- Очистите кэш: `flutter clean`
- Проверьте ошибки компиляции
